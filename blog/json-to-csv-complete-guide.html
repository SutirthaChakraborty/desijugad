JSON to CSV: the complete guide for 2026

13 min read · Updated April 2026

JSON is everywhere. REST APIs return it. NoSQL databases store it. Config files use it. But when your product manager asks for a spreadsheet, or your data analyst wants to open results in Excel, JSON's tree structure becomes a frustration fast. This guide covers every practical method to convert JSON to CSV — from a no-code browser tool to Python scripts for complex nested structures.

Why JSON and CSV are fundamentally different shapes

JSON (JavaScript Object Notation), defined in RFC 8259, is a hierarchical format: objects nest inside objects, arrays contain arrays. CSV is a flat grid: rows and columns. This mismatch is the core challenge — you can always go from flat to nested, but going from nested to flat requires decisions about what to do with depth.

A simple JSON array converts trivially:

[
  {"name": "Priya", "age": 28, "city": "Mumbai"},
  {"name": "Rohan", "age": 34, "city": "Bengaluru"}
]

Becomes:

name,age,city
Priya,28,Mumbai
Rohan,34,Bengaluru

Nested objects and arrays are where it gets interesting.

Method 1: Browser tool (fastest, no code)

For most use cases — API responses, exported data, configuration — a browser-based converter handles the job in under 10 seconds. Paste your JSON or drop the file, and the tool flattens it into CSV columns, which you can open directly in Excel or Google Sheets.

Our JSON to CSV converter runs entirely in your browser. No upload, no account. Handles nested keys with dot-notation flattening (e.g., address.city becomes a column header) and arrays by index (tags.0, tags.1).

Method 2: Python (pandas)

For automation, large files, or complex transformations, pandas' json_normalize is the standard approach:

import pandas as pd
import json

with open("data.json") as f:
    data = json.load(f)

# Flat array of objects — simplest case
df = pd.DataFrame(data)
df.to_csv("output.csv", index=False)

# Nested objects — use json_normalize
df = pd.json_normalize(data, sep=".")
df.to_csv("output.csv", index=False)

The sep="." argument controls how nested keys are joined into column names. For deeper nesting, you can use the record_path and meta parameters to extract specific nested arrays while keeping parent fields as additional columns — see the full pandas docs.

Method 3: JavaScript (Node.js)

For server-side conversion or build pipeline use, the popular json2csv npm package is the community standard:

const { Parser } = require("json2csv");
const fs = require("fs");

const data = JSON.parse(fs.readFileSync("data.json", "utf8"));
const parser = new Parser();
const csv = parser.parse(data);
fs.writeFileSync("output.csv", csv);

For browser environments, the same library works via CDN. MDN's JSON.parse reference is the canonical source for the underlying API.

Method 4: Excel Power Query

Excel 2016+ and Microsoft 365 have built-in JSON import via Power Query (Data → Get Data → From File → From JSON). This is zero-code and handles nested data with a visual column picker. The limitation: Power Query can be slow on files over ~50,000 rows, and the auto-detected column types sometimes need manual correction.

Google Sheets has equivalent import via Extensions → Apps Script, where you can paste a short script to fetch and parse JSON from a URL.

Handling the hard cases

Nested objects

The universal approach is dot-notation flattening: {"address": {"city": "Delhi"}} becomes a column called address.city. This preserves information without losing structure. Most tools and libraries do this automatically.

Arrays inside objects

Arrays require a decision: do you want one row per array item (long format) or one column per array index (wide format)?

  • Wide: tags.0, tags.1, tags.2 — columns. Works when arrays have fixed length.
  • Long: Repeat the parent row for each array item. Works for data analysis (joins, pivot tables).

Missing fields

Real-world JSON is inconsistent. One record might have an email field, another might not. Good converters produce an empty cell for missing fields; bad ones crash or skip the row. Check your output row count against input.

Special characters in values

Commas, quotes, and newlines inside values need proper CSV escaping. Per RFC 4180 (the CSV standard), fields containing these characters must be wrapped in double quotes, and any embedded double quotes must be doubled. Reliable converters handle this for you; buggy ones produce broken CSV that Excel silently misinterprets.

Alternatives to CSV for structured data

CSV is the most universally supported format, but it's not always the right choice:

  • Excel (.xlsx): Better for formatted reports. Use Desi Jugad's CSV to Excel tool after converting.
  • YAML: Human-readable config format. Use JSON to YAML for DevOps workflows.
  • Parquet: Columnar binary format for big data. Convert via pandas or Apache Arrow.

Other popular JSON-to-CSV tools (with caveats)

You'll find several online converters in search results. The most established are:

  • ConvertCSV.com — server-based, handles complex nesting, ad-supported.
  • JSON-CSV.com — server-based, freemium, good for large files.
  • CSVJSON — open-source, server-based, handles most cases well.

All three upload your data to their servers. For internal business data, API keys embedded in JSON responses, or personally identifiable information, prefer a browser-only solution.
