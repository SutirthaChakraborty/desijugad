Are online password generators safe? The cryptographic truth

12 min read · Updated April 2026

You type "random password generator" into Google. A dozen tools appear. You click one, get a 16-character string, copy it, use it for your banking login. Should you trust that password?

The answer depends almost entirely on one thing: where the randomness comes from. Get that right and online generators are perfectly safe. Get it wrong and you're using a predictable password while believing it's secure.

What makes a password "random"

Randomness in computing is harder than it sounds. Computers are deterministic — given the same inputs, they produce the same outputs. True randomness requires gathering unpredictable physical entropy: mouse movements, keyboard timing, hardware interrupts, thermal noise.

The operating system collects this entropy into a pool and exposes it via a cryptographically secure pseudo-random number generator (CSPRNG). On Linux/macOS, this is /dev/urandom or /dev/random. On Windows, it's CryptGenRandom / BCryptGenRandom. These are the gold standard for password generation.

The browser equivalent is crypto.getRandomValues(), specified in the W3C Web Cryptography API. It draws from the operating system's CSPRNG. This is the function that a trustworthy browser-based password generator must use.

A generator using Math.random() instead is cryptographically broken. JavaScript's Math.random() is not seeded from hardware entropy, is not designed for security, and may be predictable to an attacker who can observe a few outputs.

Server-side generators: the real risk

A generator that sends a request to a server and receives a password back has a critical problem: the server sees your password. Even if the connection is HTTPS, the operator of that server has a log of every password they've generated for you. Whether they retain it, whether their database is secure, and whether they're honest are all unknowns.

For most purposes — creating a password you're about to save in a password manager anyway — this risk is theoretical. But for high-value accounts, the right model is generation that happens entirely on your device.

Browser-based generators: how to verify them

A browser-based generator can be safe even though it's a website, because the generation happens in your tab, not on a server. The simplest verification:

  1. Open the generator.
  2. Disconnect from the internet (turn off WiFi or use browser DevTools → Network → "Offline").
  3. Generate passwords. If it still works, no server call is being made — the generation is genuinely local.

NIST guidelines: what a "strong" password actually means

The NIST Special Publication 800-63B is the US government's authoritative guidance on digital identity and passwords. The 2024 revision made waves by reversing several long-held rules:

  • Length matters more than complexity — a 16-character lowercase-only password has more entropy than a 10-character "complex" one with special characters.
  • Regular forced resets are counterproductive — they cause users to choose predictable patterns ("Password1!" → "Password2!"). NIST now recommends resetting only when compromise is suspected.
  • Character requirements can reduce security — rules like "must include a symbol" push users toward predictable substitutions (@ for a, 3 for e). NIST recommends allowing all printable ASCII without mandating specific character classes.
  • Minimum length should be 8; recommended is 15+ — for generated passwords saved in a manager, aim for 20+ characters.
The most important thing about a password is its entropy — the number of bits of randomness. Length from a truly random source beats complexity from a predictable pattern every time.

Password entropy: how to calculate it

Entropy (in bits) = log₂(character_set_size) × password_length

  • Lowercase only (26 chars), 12 length: log₂(26) × 12 ≈ 56 bits
  • Mixed case + digits (62 chars), 12 length: log₂(62) × 12 ≈ 71 bits
  • Full printable ASCII (95 chars), 16 length: log₂(95) × 16 ≈ 105 bits
  • Full printable ASCII, 20 length: ≈ 131 bits

128 bits of entropy is effectively unbrutable with current technology. A 20-character password from the full ASCII character set is beyond the reach of any attacker. Modern password crackers handle ~100 billion guesses per second; cracking a 128-bit-entropy password at that rate would take longer than the age of the universe.

Best free password managers to pair with a generator

A generated password is only useful if you can remember it — which means a password manager. The most trusted options:

  • Bitwarden — open-source, free tier is excellent, self-hostable. Has a built-in generator. Audited by independent security firms annually.
  • 1Password — best UX, family plan is good value, generates passwords natively. Not free but highly regarded.
  • LastPass — popular but suffered a significant breach in 2022. If you use it, ensure your master password is strong and 2FA is enabled. The breach was of encrypted vaults, so strong master passwords protected users.
  • KeePass / KeePassXC — local-only, fully offline, open-source. The gold standard for the paranoid. Requires more setup.

Check Have I Been Pwned (by security researcher Troy Hunt) to see if any of your email addresses appear in known data breaches. The Pwned Passwords endpoint lets you check specific passwords against 800 million+ compromised passwords without sending the full password — it uses a k-anonymity model over the SHA-1 hash prefix.

Passphrase vs random password

For passwords you need to type rather than paste (system login, password manager master password), consider a passphrase — 4–5 random words from a large wordlist. The original method is Diceware, which uses physical dice to select words from a 7,776-word list.

"correct horse battery staple" (the famous XKCD example) has ~44 bits of entropy — not enough in 2026. Four words from a 7,776-word list gives 51 bits; five words gives 64 bits; six words gives 77 bits. For a master password, use 6+ random words.

The bottom line

  • ✅ Safe: Browser generators using crypto.getRandomValues(), verified to work offline.
  • ✅ Safe: Password manager built-in generators (Bitwarden, 1Password, KeePass).
  • ⚠️ Use with caution: Server-side generators — fine for low-stakes passwords, avoid for banking/email/SSO.
  • ❌ Unsafe: Any generator using Math.random(). Any generator with predictable output.
