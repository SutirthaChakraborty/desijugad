Types of Barcodes and When to Use Each

Choosing the wrong barcode format causes real problems — scanners reject incompatible formats, retail POS systems expect specific standards, and some formats simply can't encode the data you need. Here's what each format is actually for:

EAN-13

European Article Number, 13 digits. The universal product identifier for retail globally — used on almost every consumer product sold in shops outside North America. The 13 digits encode a country code (first 2–3 digits), manufacturer code, product code, and a check digit calculated from the others. For India, the GS1 country prefix is 890. If you're selling products in Indian retail, your EAN-13 should start with 890.

UPC-A

Universal Product Code, 12 digits. The dominant retail barcode in the US and Canada. Functionally very similar to EAN-13 — in fact, EAN-13 is a superset of UPC-A. Most modern scanners read both. If you're selling into US retail chains, you'll need a UPC-A registered through GS1 US.

Code 128

A high-density, variable-length alphanumeric barcode. Unlike EAN/UPC which are numeric only, Code 128 encodes the full ASCII character set — letters, numbers, and symbols. Used extensively in logistics, shipping labels, and inventory management. If you need to encode a SKU like "INV-2026-A14", Code 128 is the right format.

QR Code

A 2D barcode with far more data capacity than any linear (1D) barcode. Stores up to 4,296 alphanumeric characters or 7,089 numeric characters. Used for URLs, contact info (vCard), Wi-Fi passwords, UPI payment links, and event check-ins. Critically: scannable from a phone camera with no dedicated scanner needed — which makes QR codes the right choice for consumer-facing applications.

Code 39

An older alphanumeric barcode, simpler than Code 128 but less dense. Encodes A–Z, 0–9, and a handful of special characters. Still in use in automotive parts tracking, military supply chains, and some healthcare systems that haven't migrated to Code 128.

ITF-14

Used on shipping cartons and outer packaging — not on individual retail items. Encodes a 14-digit GTIN (Global Trade Item Number). If you're packing products into cases for distribution, ITF-14 goes on the case; EAN-13 goes on the individual product inside.

Data Matrix

A 2D code that can be printed extremely small while still encoding substantial data. Used in electronics manufacturing (printed directly on PCBs), pharmaceutical packaging (unit-level drug tracking), and postal services. Requires a 2D scanner or smartphone camera — a standard 1D laser barcode scanner cannot read Data Matrix.

Getting a Real UPC/EAN Number

A barcode generated by any tool — including ours — is only a valid, scannable identifier in the real world if the number behind it is properly registered. The rules depend on your use case:

  • Retail products (supermarkets, Amazon, Flipkart): You must purchase a GS1 company prefix from GS1, the global barcode authority. In India, that's GS1 India (gs1india.org). The prefix is what makes your barcode globally unique and scannable at any POS system in the world. Counterfeit or duplicate barcodes in retail cause serious point-of-sale errors and can result in chargebacks, delisting, or legal issues.
  • Internal inventory and asset tracking: Any number works. You control the database that maps barcode numbers to items, so there's no registration requirement. Our generator is perfectly suited for this use case.
  • Event ticketing and access control: Your ticketing system assigns the numbers. The barcode is just a machine-readable version of an ID in your own system.

Barcode Print Requirements

A barcode that looks fine on screen can fail to scan when printed if these requirements aren't met:

  • Minimum bar width: 0.8mm for the narrowest bar. Below this, most laser scanners fail to reliably resolve the bars.
  • Quiet zones: Mandatory white space on the left and right of the barcode (and top/bottom for 2D codes). Without quiet zones, scanners can't find the start/end of the code. EAN-13 requires a minimum quiet zone of 3.63mm on each side.
  • Minimum resolution: 300 DPI for print. 600 DPI preferred for small labels. Screen resolution (72–96 DPI) is completely unsuitable for barcode printing.
  • Color: Black bars on white background is the standard. Other colour combinations can work — dark bars on light background — but must maintain sufficient contrast for scanner optics. Avoid red bars (laser scanners use red light and can't see red bars).
  • Always test-scan before mass printing. Print a test label and scan it with the actual scanner that will be used in production — handheld scanner, POS terminal, or phone camera — before printing a full run.

Frequently Asked Questions

What is the difference between a barcode and a QR code?

Traditional barcodes (EAN-13, Code 128, UPC-A) are 1D — they encode data in a single horizontal dimension using parallel bars of varying widths. They store limited data, typically just numbers or short alphanumeric strings. QR codes are 2D — they encode data in both horizontal and vertical dimensions using a matrix of squares. This allows far more data capacity (up to 7,089 numeric characters vs. a few dozen for a 1D barcode). QR codes also have built-in error correction, meaning they can still be read even if up to 30% of the code is damaged or obscured.

Can I use a generated barcode for selling products in stores?

Not without registration. For products to be scanned at retail POS systems (supermarkets, online marketplaces like Amazon or Flipkart), the barcode number must be registered with GS1. Any number you generate that isn't a registered GS1 prefix could conflict with another company's products and cause POS errors. For internal use — your own warehouse, your own inventory system — you can use any number you like. For retail, purchase a GS1 company prefix from GS1 India (gs1india.org) or GS1 US.

What file format should I use for barcode images?

Use SVG for anything going to print. SVG is a vector format — it scales to any size without losing sharpness, which is critical for barcodes where bar-edge precision affects scannability. Use PNG if you need to embed the barcode in a Word document, Excel sheet, or email, and export it at 300 DPI minimum (600 DPI preferred for small labels). Never use JPEG for barcodes — JPEG compression introduces artefacts that blur bar edges and can cause scan failures.
