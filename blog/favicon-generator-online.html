Favicon Generator Online Free — Create Website Icons Instantly

April 27, 2026 • 4 min read

A favicon is the tiny icon that appears in browser tabs, bookmarks, address bars, and increasingly in Google search results next to your site name. Despite being as small as 16×16 pixels, it's often the first visual signal users associate with your site. A missing favicon signals an unfinished site. A blurry or wrong-size favicon signals a lack of attention to detail. A sharp, recognizable favicon signals professionalism before the user has read a single word.

All the Sizes You Actually Need

The frustrating reality of favicons is that there's no single file that works everywhere. Different browsers, operating systems, and devices expect different formats and sizes. Here's what a complete favicon setup looks like in 2026:

  • favicon.ico: A multi-resolution ICO file containing 16×16, 32×32, and 48×48 pixel versions combined into a single file. Still needed for legacy Internet Explorer compatibility and for some email clients that display sender favicons. Place it at the root of your domain (/favicon.ico) — browsers request it automatically even without an HTML link tag.
  • favicon-16x16.png: The default browser tab size. Small, but the most commonly displayed version.
  • favicon-32x32.png: Standard resolution for modern browsers and Retina displays at 1×.
  • apple-touch-icon.png (180×180px): Used when someone adds your site to their iOS home screen as a bookmark. Also used by Safari for pinned tabs. Without this, iOS uses a screenshot of your page instead — which looks terrible.
  • android-chrome-192x192.png and android-chrome-512x512.png: Required for Chrome on Android and for Progressive Web App (PWA) manifests. The 512px version is used for the PWA splash screen when your app is loading.
  • Safari pinned tab SVG: A monochrome, single-color SVG used when users pin your tab in Safari on macOS. Must be maskable (solid shape, no transparency tricks).
  • og-image.png (1200×630px): Not technically a favicon, but closely related — this is the image shown when your link is shared on social media (Twitter, Facebook, LinkedIn, WhatsApp). It's referenced via <meta property="og:image">.

The HTML to Link Your Favicons

Add these tags inside your <head> section. Modern browsers use the PNG files and only fall back to .ico for legacy support:

<link rel="icon" type="image/png" sizes="32x32" href="/favicon-32x32.png">
<link rel="icon" type="image/png" sizes="16x16" href="/favicon-16x16.png">
<link rel="apple-touch-icon" sizes="180x180" href="/apple-touch-icon.png">
<link rel="manifest" href="/site.webmanifest">
<meta name="theme-color" content="#ffffff">

The site.webmanifest file is a small JSON file that references your Android Chrome icons and sets your PWA display preferences. The theme-color meta tag controls the browser UI color on mobile Chrome.

SVG Favicons — The Modern Approach

Since 2021, Chrome, Firefox, Edge, and Safari all support SVG favicons. An SVG favicon is infinitely scalable — it looks crisp on any display resolution with a single file. You can also embed CSS inside the SVG, which enables two powerful features: CSS animations (though use these sparingly) and dark mode support.

To add dark mode support to an SVG favicon, include a prefers-color-scheme media query inside the SVG's <style> block. This lets your icon automatically switch between a light and dark version based on the user's OS theme — a detail that polished sites get right and most don't bother with.

Add it alongside your PNG fallbacks: <link rel="icon" href="/favicon.svg" type="image/svg+xml">. Browsers that support SVG will use it; older browsers fall back to the PNG and ICO versions.

Design Principles for a Favicon That Works

The hardest constraint in favicon design is the 16×16 pixel floor. At that size, fine details, thin strokes, and multiple colors collapse into visual noise. Here's what actually works:

  • Simplify aggressively. A letter, a single icon, or a bold shape works. A detailed illustration does not.
  • One or two colors maximum. High contrast between foreground and background is essential — light icon on dark background, or vice versa.
  • Test against both browser themes. Chrome's dark mode shows favicons on a dark tab bar. Your white-background favicon will disappear entirely.
  • Avoid fine details. Thin lines, small text, and gradients all disappear at 16×16px. If you can't draw it with a marker in five strokes, it's too complex.
  • Square format. Don't fight the shape. Favicons display in a square (or circle on some platforms). Landscape or portrait designs get awkwardly cropped.
  • A favicon is a logo mark, not a wordmark. Your full company name will not be legible. Use the icon portion of your logo, or a stylized initial.

Google Search and Favicons

Google displays favicons next to page titles in mobile search results. Getting this right gives your result a visual edge in crowded SERPs. Google's requirements: the favicon must be visible on a white background, at least 48×48px, and square. Google crops non-square favicons into a circle, so designs with important content near the edges may lose it. Use a PNG or SVG — Google's crawler handles these better than ICO. Google may take a few days to pick up a newly added or changed favicon, and it can take longer if your site is new.

Frequently Asked Questions

What is the difference between favicon.ico and favicon.png?

favicon.ico is the original format — a container file that can hold multiple resolutions (16×16, 32×32, 48×48) in a single file. Browsers have supported it since the late 1990s and will look for it at /favicon.ico automatically. favicon.png is a standard image file at a single resolution. Modern browsers (Chrome, Firefox, Edge, Safari) prefer PNG or SVG over ICO. In practice, you should provide both: the .ico file for legacy compatibility and PNG files linked via <link> tags for modern browsers.

Why isn't my updated favicon showing?

Browsers cache favicons aggressively — often for days. To force a refresh: 1) Hard reload the page with Ctrl+Shift+R (Windows) or Cmd+Shift+R (Mac). 2) Clear your browser cache entirely. 3) Open the favicon URL directly (yoursite.com/favicon.ico) and hard reload that. 4) In Chrome DevTools, go to the Network tab, right-click the favicon request, and select "Clear browser cache." If none of these work, append a version query string to your favicon link tag: href="/favicon.png?v=2" — this forces the browser to treat it as a new file.

Do I need all these favicon sizes?

For a basic website, a favicon.ico at the root plus a 32×32 PNG linked in your HTML covers most browsers. For a complete, professional setup — especially if you want iOS home screen icons, Android PWA support, and Google search favicons — you need the full set. If you're building a PWA, the 192×192 and 512×512 PNG files in your site.webmanifest are mandatory.
