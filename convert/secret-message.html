Best practices

  • Use a unique passphrase for each highly sensitive file or message.
  • Share the encrypted payload and the passphrase over different channels.
  • Remember that losing the passphrase means losing access to the content.
  • Download encrypted files if you want a portable offline copy.

When should I encrypt text versus a file?

Use secret text for a short note, one-time password hint, or credentials bundle you want to paste into a messenger. Use secret file when you want to protect a document, screenshot, ZIP, or any other attachment before sharing it.

What is actually stored in the encrypted payload?

The payload contains the algorithm name, random salt, random IV, iteration count, and encrypted bytes. It does not store your passphrase.

Related security tools

  • Generate file and text hashes to verify integrity before or after you share encrypted payloads.
  • Create a strong password if you need help choosing a safer passphrase structure.
  • Encode files with Base64 when you need text-safe packaging rather than encryption.
  • Clean up or hash text before encrypting notes, snippets, or developer secrets.

FAQ

Is this safe for secret messages?

It is a solid privacy-first browser tool for everyday sharing, powered by modern Web Crypto primitives. For highly regulated or organizational workflows, use your approved security tooling and policies.

What happens if I forget the passphrase?

The content cannot be recovered. The passphrase is never stored or transmitted, so there is no fallback recovery.

Can I encrypt any file type?

Yes. The tool preserves the original filename and MIME type inside the encrypted payload, then restores them when decrypting.
