Tip — saving any webpage as PDF: Open the page in Chrome/Edge/Firefox, press Ctrl+P (Windows/Linux) or Cmd+P (Mac), change the destination to Save as PDF, and click Save. This uses the browser's native PDF renderer for perfect fidelity.

When to use HTML-to-PDF conversion

HTML-to-PDF is useful when you want to archive a webpage you've built, create a printable version of an email template, save a report generated by a web app, or share a designed document as a portable PDF file.

How this tool works

Your HTML is injected into a sandboxed <iframe> using srcdoc. After it loads, html2canvas captures the rendered iframe as a canvas element. jsPDF then embeds the canvas as a JPEG image scaled to the chosen page size. The result is a PDF where each page is an image snapshot of your HTML.

Limitations

  • External resources (images from other domains, external CSS) may not load due to CORS restrictions in the sandbox
  • Web fonts loaded from Google Fonts or similar CDNs may fall back to system fonts
  • CSS animations and transitions are captured at their initial state
  • Very long HTML pages are captured as one tall page rather than multi-page PDF

FAQ

Why do my images not show in the PDF?

External images are blocked by browser CORS policy when loaded in a sandboxed iframe. Embed images as Base64 data URIs in your HTML for reliable rendering: <img src="data:image/png;base64,...">

Can I use this to convert a receipt or invoice template?

Yes — paste your invoice HTML (with inline CSS for best results), preview it, and click Save as PDF. This is a great use case for this tool.

How is this different from browser Print > Save as PDF?

Browser Print uses a dedicated print rendering engine that handles pagination, @media print CSS, and page breaks. html2canvas captures a pixel snapshot. For most HTML documents, browser Print gives better multi-page results.

Does this work on mobile?

Yes — html2canvas and jsPDF both work on mobile browsers. The output quality is the same as on desktop.
