About the PNG format

PNG (Portable Network Graphics) was created in 1995 as a free, open alternative to the GIF format. Its most important property is lossless compression — when you save a PNG, not a single pixel of data is changed. You can save, edit and re-save a PNG any number of times with zero quality degradation.

PNG supports full alpha-channel transparency, which allows each pixel to have any level of opacity. This makes it the standard choice for logos, icons, UI elements, and any graphic that must be placed on different coloured backgrounds. The trade-off is file size: PNG files are significantly larger than JPG for photographic content.

PNG supports bit depths from 1-bit (black/white) up to 48-bit true colour with 16 bits per channel, making it suitable for high-dynamic-range editing workflows. For web delivery of photos, consider WebP which offers lossless mode with better compression than PNG.

FAQ

Does PNG support transparent backgrounds?

Yes. PNG supports full alpha-channel transparency, meaning each pixel can have its own opacity level from fully transparent to fully opaque. This makes PNG the standard choice for logos, icons, and any graphic that needs to sit on a coloured background. JPG does not support transparency.

Why is my PNG file so much larger than JPG?

PNG uses lossless compression, which preserves every pixel exactly. JPG uses lossy compression that discards subtle details to achieve much smaller files. For photographic images, JPG files are typically 5–10x smaller than PNG at comparable visual quality. PNG is best for screenshots, logos, and graphics — use JPG or WebP for photos.

How do I convert PNG to JPG without losing transparency?

JPG does not support transparency. When converting PNG to JPG, transparent areas are filled with a background colour — usually white. If you need to preserve transparency, convert to WebP or keep the file as PNG.
