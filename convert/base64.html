What Base64 is, and when you need it

Base64 encoding represents binary data using only 64 printable ASCII characters (A–Z, a–z, 0–9, +, /). It's how images are embedded directly in HTML/CSS (data:image/png;base64,...), how email attachments travel, how some APIs carry binary payloads, and how credentials are encoded in HTTP Basic Auth headers.

When to use URL-safe Base64

Standard Base64 uses + and /, which have special meaning in URLs and file paths. URL-safe Base64 (used by JWT tokens and many modern APIs) swaps them for - and _.

FAQ

Can I encode large files?

Yes — we use FileReader, which handles files up to your browser's memory limit (usually 1–2 GB).

Is my input sent anywhere?

No. All encoding/decoding happens with native browser APIs (btoa/atob and FileReader).

Can I decode Base64 back into a file?

Yes — paste Base64 into the input, switch to "Decode", and click Download to save the decoded bytes.
