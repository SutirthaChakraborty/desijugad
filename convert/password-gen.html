How are these passwords generated?

This tool uses window.crypto.getRandomValues() — the Web Cryptography API. This is a cryptographically secure pseudorandom number generator (CSPRNG), not the insecure Math.random(). It's the same API used by password managers and HTTPS connections.

Password entropy guide

  • Below 40 bits — Weak. Vulnerable to brute force.
  • 40–60 bits — Moderate. OK for low-stakes accounts.
  • 60–80 bits — Strong. Fine for most personal accounts.
  • 80–100 bits — Very strong. Suitable for sensitive accounts.
  • 100+ bits — Excellent. Suitable for master passwords.
