How PDF compression works

PDFs often contain high-resolution images that take up most of the file size. This tool uses pdf-lib and the Canvas API to decode embedded JPEG images, re-encode them at a lower quality, and embed them back — resulting in a smaller file. Text, vector graphics, and fonts are not affected.

Note: PDFs that consist entirely of text and vector graphics (no images) will not compress significantly.

FAQ

What's the maximum file size?

There's no hard limit, but large PDFs (50MB+) may take 30–60 seconds to process. The browser does all work in memory.

Are scanned PDFs (image-only PDFs) supported?

Yes — scanned PDFs consist entirely of images, so they compress well. For text extraction from scanned PDFs, try the OCR tool.
