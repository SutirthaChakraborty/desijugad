About 3GP audio extraction

3GP files contain video (H.263 or H.264) and audio (AMR or AAC) tracks. This tool attempts to decode the audio track using the browser's built-in Web Audio API decoder. If the browser supports the codec, the audio is extracted and re-encoded as MP3 using lame.js.

Common 3GP audio codecs:

  • AAC audio — Well supported in Chrome and Safari. Best chance of working.
  • AMR-NB audio — Limited browser support. May not decode in desktop browsers. Try Chrome on Android.
  • AMR-WB audio — Even more limited. Consider the AMR to MP3 tool for AMR files.

FAQ

My 3GP file says "decoding failed". Why?

The most common reason is that your 3GP file uses AMR audio encoding, which desktop browsers don't support natively. Try Chrome on an Android phone, or convert using a desktop tool. Files with AAC audio in the 3GP container usually work fine.

What is the 3G2 format?

3G2 is very similar to 3GP — both are based on MPEG-4 Part 12. 3G2 was standardized by 3GPP2 (CDMA carriers) while 3GP was for GSM carriers. This tool accepts both .3gp and .3g2 files.

The audio quality sounds bad. Can I improve it?

3GP was designed for low-bandwidth mobile transmission. The original audio is often low quality (8 kHz AMR). Higher MP3 bitrates won't improve a low-quality source — the input quality is the ceiling. 64 or 128 kbps is sufficient for 3GP audio.
