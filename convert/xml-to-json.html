XML to JSON (and back) — a practical guide

      
XML and JSON are both data interchange formats. JSON is simpler and more common in modern web APIs. XML remains popular in enterprise systems, configuration files, SOAP APIs, and RSS/Atom feeds.


      
How attributes are handled

      
XML attributes (e.g. <person id="42">) are converted to a key with the prefix you set (default: @_). So the above becomes { "person": { "@_id": "42" } }.


      
Repeated elements become arrays

      
If multiple sibling elements share the same tag name, they are automatically collected into a JSON array.


      
FAQ

      

        Does this handle large XML files?
        
Yes — the DOMParser in the browser can handle multi-megabyte XML files. Tested with 10 MB+ configs without issues.

      

      

        What about XML namespaces?
        
Namespace prefixes are preserved as part of the key name (e.g. ns:element → "ns:element"). Namespace declarations (xmlns:ns=…) are treated as attributes.
