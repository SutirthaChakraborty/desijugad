OTF vs TTF — what is the difference?

Both OTF (OpenType Font) and TTF (TrueType Font) are widely supported font formats used for desktop and web applications. Here is how they differ:

TTF (TrueType Font) was created jointly by Apple and Microsoft in the late 1980s as an alternative to Adobe's PostScript Type 1 fonts. TrueType uses quadratic Bezier curves to describe glyph outlines. TTF was bundled with Windows 3.1 and became the standard for desktop fonts through the 1990s and 2000s. TTF fonts are supported universally — every operating system from Windows 95 onwards handles them natively.

OTF (OpenType Font) was developed jointly by Adobe and Microsoft in the mid-1990s as a successor to both TTF and PostScript Type 1. OpenType is a superset of TrueType — it can contain either TrueType outlines (quadratic) or PostScript/CFF outlines (cubic Bezier). The key advantage of OTF is its support for advanced typographic features through OpenType Layout Tables: ligatures, small caps, old-style figures, contextual alternates, stylistic sets, swashes, and more. Professional typefaces designed for editorial and print use are typically distributed as OTF.

WOFF / WOFF2 are compressed wrappers around OTF or TTF designed for efficient web delivery. WOFF2 uses Brotli compression and is 30% smaller than WOFF. For web use (CSS @font-face), always use WOFF2.

FAQ

What is the difference between OTF and TTF?

Both are font formats storing glyph outlines and metrics. TTF (TrueType) uses quadratic Bezier curves and was created by Apple and Microsoft in the 1980s. OTF (OpenType) was created by Adobe and Microsoft in the 1990s as a superset of TTF — it adds advanced typographic features like ligatures, small caps, stylistic sets, and contextual alternates. For most screen uses, both look virtually identical. OTF is preferred for professional typography; TTF has slightly broader legacy compatibility.

What is WOFF and when should I use it?

WOFF (Web Open Font Format) is a compressed wrapper around TTF or OTF data, designed specifically for web delivery. WOFF2 uses Brotli compression and reduces font file sizes by about 30% compared to WOFF. For web use, always serve fonts as WOFF2 with WOFF as a fallback. TTF and OTF are for desktop installation; WOFF/WOFF2 are for CSS @font-face rules.

Can I use any font I download on my website?

Only if the font license permits web embedding. Many fonts are licensed for desktop installation only and prohibit embedding in websites or web font conversion. Always check the font's license file (usually SIL OFL, Apache 2.0, or a commercial license). Google Fonts and fonts under SIL OFL are generally safe to use on the web.
