Why convert between JSON and YAML?

Both formats describe the same structures — objects, arrays, strings, numbers, booleans, null — but YAML is friendlier for humans to read and edit, while JSON is friendlier for machines and APIs. DevOps pipelines, Kubernetes manifests, Docker Compose files, GitHub Actions workflows, and Ansible playbooks all live in YAML. REST APIs, npm configs, and most CLIs prefer JSON. This tool flips between them cleanly.

What it handles

  • Arbitrary nesting — arrays of objects of arrays, no depth limit.
  • Multi-line strings, special characters, and escaped quotes.
  • Correct YAML indentation (2, 4, or 8 spaces).
  • Booleans, numbers, and null values preserved in both directions.

FAQ

Does my data get uploaded?

No. This runs entirely in your browser using the js-yaml library. Your configs stay on your machine.

Will YAML comments be preserved when going to JSON?

JSON doesn't support comments, so YAML comments are dropped when converting YAML → JSON. Going the other way, there are no comments to preserve.

Can I convert huge config files?

Files up to a few megabytes convert in under a second. Larger files may take a moment but will still work.
