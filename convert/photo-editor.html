A proper photo editor that runs entirely in your browser

Most browser-based "photo editors" do one thing — resize, or compress, or convert. Desi Jugad Photo Editor combines every basic editing operation into a single tab: visual crop with handle dragging, resize with aspect lock, rotate and flip, brightness/contrast/saturation/sharpness adjustment, platform export presets, and a plain-English command bar. Everything uses the browser's Canvas API — no uploads, no server, no account.

The Ask Me Anything bar

Type what you want in plain English and click Apply. The command parser understands hundreds of phrase variations for common operations: "crop to square", "make it 1080 wide", "rotate 90 clockwise", "flip horizontal", "export as webp 80%", "brightness +30", "compress under 200kb", "Instagram post", "passport photo". No AI, no API call — pure JavaScript pattern matching, runs instantly.

Platform presets

Click any preset to immediately resize and/or crop your image to that platform's recommended dimensions. Presets include Instagram (square, portrait, stories), Facebook (post, cover), LinkedIn post, Twitter/X (post, header), A4 print at 300 DPI, passport photo, and common web sizes. The preset crops to the aspect ratio first, then resizes — so your image fits correctly without stretching.

Adjustments

Brightness, contrast, saturation, and sharpness are applied as real-time CSS filter previews. When you click Bake adjustments, the filters are rendered permanently into the canvas using pixel-level manipulation — meaning they'll export correctly and you can continue editing on top.

Undo / redo

Every operation (crop, resize, rotate, flip, bake) pushes a snapshot to the history stack. Use the ↩ Undo and ↪ Redo buttons or Ctrl+Z / Ctrl+Y (Cmd+Z / Cmd+Y on Mac). Up to 20 history steps are kept.

Export options

Export as JPEG (smallest files), PNG (lossless, keeps transparency), or WebP (best compression with quality). Use the quality slider to balance file size vs. sharpness. The DPI field embeds a resolution tag in the file metadata — useful for print: set 300 dpi for any image going to a printer.

Frequently asked questions

Is my photo uploaded to any server?

No. All processing uses your browser's Canvas API and JavaScript. Your image never leaves your device. There are no analytics on the image content. Close the tab and nothing remains.

What's the maximum image size?

Up to 50 MB and up to approximately 16,000 × 16,000 pixels. Very large images (above 6000px on a side) may be slow to process on low-end devices because each operation redraws the canvas. For batch processing, use the Image Resize tool which handles up to 100 files.

Can I use this for batch editing multiple images?

This tool is optimised for single-image detailed editing. For batch resizing (same settings applied to many files), use the Image Resize + DPI tool. For batch compression, use the Image Compressor.

Does the crop keep the full resolution?

Yes. The crop operates on the full-resolution canvas, not on the scaled preview. The cropped output is the exact pixels from your original image at the dimensions you see in the info bar.

What does "Bake adjustments" mean?

Brightness, contrast, saturation, and sharpness are shown as CSS filter previews while you're adjusting. "Bake" renders them permanently into the pixel data of the canvas, so when you export or do further edits (crop, resize) the adjustments are preserved. If you skip Bake and export, the adjustments are still applied to the exported file.

What does the DPI selector do?

It embeds a resolution tag (pHYs chunk for PNG, JFIF density for JPEG) in the exported file. This tells print software how large to print the image. It does not change the pixel count — a 1200×630 image at 300 DPI is still 1200×630 pixels, but a printer will print it at ~4×2.1 inches instead of ~16.7×8.75 inches.
