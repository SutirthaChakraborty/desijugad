What is a file hash?

A hash is a fixed-length fingerprint of a file or text block. Change even one bit and the output changes. That makes hashes useful for verifying downloads, comparing files, or storing integrity checks in documentation.

What’s the difference between SHA-1, SHA-256, SHA-384, and SHA-512?

SHA-1 is legacy and still seen in older tooling, but it should not be used for new security-sensitive workflows. SHA-256 is the most common checksum format for releases. SHA-384 and SHA-512 produce longer digests and are sometimes used in stricter environments.

FAQ

Are the hashes generated locally?

Yes. This tool uses the browser’s built-in Web Crypto API and never uploads the text or file you provide.

Can I hash very large files?

You can hash most common files, but the browser must read the file into memory. Very large files may be limited by device memory.

Can I compare an expected checksum?

Yes. Paste a hash into the compare box and any matching result will be marked as a verified match.
