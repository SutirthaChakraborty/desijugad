How in-browser audio conversion works

Your browser's Web Audio API can decode any audio format it natively supports into raw PCM data (uncompressed samples). This tool then re-encodes that PCM data into your chosen output format:

  • WAV output — PCM data is wrapped in a standard RIFF/WAV container. Lossless, large files.
  • MP3 output — PCM is fed into lame.js, a WebAssembly port of the LAME MP3 encoder. Result is a proper VBR/CBR MP3 file.

Note: Browser audio decoding supports MP3, WAV, OGG, AAC/M4A, FLAC, and OPUS depending on your browser. WebM, MKV, and exotic formats may not decode — try Chrome for the widest support.

Trim audio while converting

Set Trim start and Trim end in seconds to extract a clip. Leave blank to convert the full file. Great for cutting intros, ringtones, or podcast clips.

FAQ

Which formats are supported as input?

Any format your browser's audio decoder supports: MP3, WAV, OGG, FLAC, AAC/M4A, OPUS. The decoder is built into the browser — no plugin needed.

Is there quality loss when converting to WAV?

When decoding to WAV, the process is lossless from the browser's perspective — you get the decoded PCM at 16-bit/44.1 kHz. If the source was MP3, the original lossy compression was already applied, but no additional quality is lost.

What is the maximum file size?

Limited only by your device's RAM. Audio is decoded into raw PCM which can be large — a 10-minute stereo track is ~100 MB in memory. Most modern devices handle this fine.
