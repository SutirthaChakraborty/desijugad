About FBX and OBJ

FBX (Filmbox) is an Autodesk proprietary format that supports full scene data: meshes, materials, lights, cameras, animation, bones, and blend shapes. It's the standard interchange format between professional 3D tools like Maya, 3ds Max, MotionBuilder, and game engines like Unity and Unreal Engine.

OBJ (Wavefront Object) is a universal 3D interchange format focused on geometry. It's supported by virtually every 3D application and is ideal when you need to share a mesh without scene data or animation.

What gets converted

This tool extracts vertex positions and polygon face indices from ASCII FBX geometry nodes. Normals, UV coordinates, materials, animations, and skeleton data are not included in the OBJ output.

For full FBX conversion

If you need complete FBX conversion with materials, animations, and textures, use Blender (free) — import FBX, then export as OBJ. Blender handles both binary and ASCII FBX with full scene support.

FAQ

How do I know if my FBX is ASCII or binary?

Open the file in a text editor (Notepad, VSCode). If the first line reads ; FBX 7.4.0 project file or similar readable text, it's ASCII FBX. If you see garbled binary characters, it's binary FBX — re-export from your 3D app in ASCII mode.

The OBJ is empty or has very few polygons

Check that your FBX contains a Geometry node with Vertices and PolygonVertexIndex arrays. Some FBX files embed geometry inside nested nodes — try exporting with a simpler hierarchy or apply transformations before export.

Can I convert binary FBX?

Not in this browser tool. Binary FBX requires the Autodesk FBX SDK or a library like three.js FBXLoader. For binary FBX, open in Blender or use the FBX SDK.

Will UV coordinates be in the OBJ?

If the FBX Geometry node contains LayerElementUV data in ASCII form, the tool will attempt to include UV coordinates as vt lines. Standard vertex UVs are supported; layered or blendshape UVs may not parse correctly.
