Font format guide

Modern fonts ship in a confusing variety of formats. Here's when to use each:

  • TTF (TrueType Font) — The classic desktop font format. Supported by every OS since the 1990s. Best for Windows desktop apps and older software.
  • OTF (OpenType Font) — A superset of TTF with support for advanced typographic features like ligatures, alternate glyphs, and multilingual support. Preferred by designers.
  • WOFF (Web Open Font Format) — A compressed version of OTF/TTF designed for use in websites via CSS @font-face. Smaller file size, same font data.
  • WOFF2 — Even more compressed than WOFF. The standard for web fonts today. Use Brotli-capable tools or Google Fonts to generate WOFF2.

FAQ

Is converted OTF/TTF safe to install on my computer?

Yes. The conversion re-serializes the same font data in a different container. The resulting font is equivalent to the original and safe to install on Windows, macOS, or Linux.

My font has special glyphs — will they survive conversion?

opentype.js parses and preserves all glyph data including ligatures, kerning tables, Unicode ranges, and OpenType feature tags. The conversion is lossless for glyph outlines.

Can I convert multiple fonts at once?

Currently one font at a time. For batch conversion, consider using pyftsubset (part of fonttools) on the command line.

What does opentype.js do?

opentype.js is an open-source JavaScript library for parsing, manipulating, and generating font files. It handles all the low-level binary parsing of OTF, TTF, and WOFF formats.
