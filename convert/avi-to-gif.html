How video to GIF works

This tool loads your video into a hidden <video> element, seeks through it frame by frame, paints each frame to a <canvas>, reads the pixel data, and feeds it to omggif — a pure JavaScript GIF encoder. The resulting animated GIF loops forever.

  • Frame capture — Frames are captured by seeking the video at precise timestamps (1/fps intervals).
  • Color quantization — GIF supports 256 colors per frame. Colors are quantized using a simple median-cut algorithm in omggif.
  • GIF size — GIFs can be large. A 5-second clip at 10 fps, 320px wide can be 2-5 MB. Reduce width or fps to shrink.

FAQ

Why is the GIF file so large?

GIF compression is inefficient compared to modern video codecs. Every frame is stored independently (unless they're identical). Reduce the output width to 320px and use 5 fps for the smallest file. For sharing on the web, consider MP4 — same visual, 10x smaller file.

My AVI file doesn't load. What should I do?

AVI is not natively supported by most browsers. Convert your AVI to MP4 first using the Video Converter, then come back here to make a GIF from the MP4.

The GIF looks blurry or has wrong colors. Why?

GIF's 256-color palette limitation causes color banding and dithering. This is a fundamental GIF limitation, not a tool bug. For sharper output, use a smaller clip (fewer frames = better palette allocation per frame).

Can I make a GIF from just part of the video?

Yes. Set the Start and End seconds in the options to clip the video before GIF conversion. A 3-5 second clip at 10 fps is the sweet spot for a good-quality, reasonable-size GIF.
