From notes to proper HTML in a heartbeat

Markdown is how most of the modern internet writes — READMEs on GitHub, blog posts on Substack, issue comments on Linear, notes in Obsidian. But when you need to paste content into a WYSIWYG editor, a CMS, or an email, you need real HTML. This tool converts Markdown to semantic, standards-compliant HTML — and shows a live preview so you can see exactly what you're getting.

Supported Markdown

  • Headings, paragraphs, line breaks (hard or soft).
  • Bold, italic, strikethrough, and inline code.
  • Ordered, unordered, and nested lists including task lists.
  • Links, images, autolinks, and reference-style links.
  • Code fences with language hints.
  • Tables (GFM-style).
  • Block quotes, horizontal rules, HTML passthrough.

FAQ

Which Markdown flavor does this use?

We use marked.js, which follows CommonMark plus GitHub-flavored extensions (tables, strikethrough, task lists, autolinks, hard line breaks).

Is the output safe to embed?

With "Sanitize HTML" on, we run the result through DOMPurify, which strips script tags, event handlers, and other XSS vectors. Turn it off only if you trust the source completely.

Does it support MathJax or Mermaid?

Not directly. This tool focuses on standard Markdown. For diagrams and equations, paste the HTML into a renderer that supports those extensions.
