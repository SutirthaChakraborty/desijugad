About the SVG format

SVG (Scalable Vector Graphics) is an XML-based vector image format developed by the W3C and first published in 1999. Unlike raster formats (PNG, JPG, WebP) which store images as a grid of pixels, SVG describes images as mathematical shapes — paths, circles, rectangles, curves, and text — defined in code.

The key advantage is infinite scalability. An SVG logo looks razor-sharp whether displayed at 16×16 pixels as a favicon or stretched across a 10-metre billboard, because the browser or rendering engine recalculates the shapes at the display resolution. File sizes for simple graphics are also very small — a logo that would be 50 KB as a PNG might be 2 KB as an SVG.

SVG supports CSS styling, JavaScript interactivity, animations, filters, and gradients. It is natively supported in all modern browsers and can be embedded directly in HTML. SVG is the recommended format for logos, icons, illustrations, charts, maps, and any graphic that needs to scale or be animated on the web.

Limitations: SVG is not suitable for photographs (use JPG or WebP instead). Complex SVGs with many paths can be slow to render. Some environments (older Office versions, certain email clients, image preview applications) do not support SVG — in those cases, export to PNG.

FAQ

What resolution should I choose when converting SVG to PNG?

It depends on the intended use. For web use at standard display resolution, 96 DPI or 2x (192 DPI) is usually enough. For print, use at least 300 DPI. For very large prints or retina displays, 600 DPI. Because SVG is vector-based, you can export at any resolution without quality loss.

Why use SVG instead of PNG for logos?

SVG files are resolution-independent — a logo in SVG looks crisp on a phone screen, a 4K monitor, and a billboard. PNG is a raster format with fixed pixels; scaling it up causes blurring. SVG files for simple logos are also typically much smaller than PNG. However, SVG is not universally supported everywhere PNG is.

Can I edit an SVG file as code?

Yes. SVG is an XML-based text format, which means you can open any .svg file in a text editor or code editor and directly read and modify the markup. This is useful for changing colours, sizes, removing elements, or adding CSS animations. Vector drawing tools like Inkscape, Adobe Illustrator, and Figma also export SVG.
