STL vs OBJ — format differences

STL (Stereolithography) is the de-facto standard for 3D printing. It stores only triangle mesh data — no colors, no materials, no UV coordinates. Every face is an independent triangle with its own normal vector.

OBJ (Wavefront Object) is a more flexible format used in 3D modelling and rendering. It supports shared vertices, UV texture coordinates, normals, and can reference external MTL material files. OBJ is widely supported in Blender, Maya, Cinema 4D, and game engines.

Binary vs ASCII STL

Binary STL is the most common — compact and fast to parse. Each triangle takes exactly 50 bytes: 12 bytes for the normal vector, 36 bytes for three vertices (12 bytes each), and 2 bytes for an attribute byte count. ASCII STL spells out coordinates as text — larger but human-readable.

OBJ output format

The generated OBJ file includes v x y z lines for each vertex position, vn x y z lines for each normal vector, and f v//vn v//vn v//vn face definitions.

FAQ

Why is the OBJ larger than the STL?

OBJ is a text format while binary STL is compact binary. A binary STL with 10,000 triangles is about 500 KB; the equivalent OBJ text file will be several times larger. The geometry is identical.

Does the converted OBJ preserve the original scale?

Yes — coordinates are written with full float precision. The scale, position, and orientation are identical to the STL source.

Can I convert OBJ back to STL?

Not yet in this tool. For reverse conversion (OBJ → STL), Blender is free and exports STL natively.

What about multi-shell STL files?

STL files may contain separate solid bodies (shells). Each solid in ASCII STL is named; binary STL has no shell names. The OBJ output will contain all triangles as a single object.
