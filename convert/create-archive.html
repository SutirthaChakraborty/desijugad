Why zip your files?

A ZIP archive bundles many files into a single .zip file, making it easy to email, upload, share, or store. Compression reduces total size, especially for text-heavy files like source code, logs, and documents.

Folder structure is preserved

When you add a folder using the folder picker, each file is stored with its relative path inside the ZIP. Extracting on any device will recreate the original folder tree — great for sending project directories.

Everything stays local

The archive is built entirely in your browser's memory using the open-source JSZip library. Nothing is uploaded. The resulting ZIP is downloaded directly to your device.

FAQ

How large an archive can I create?

Limited only by your device's available RAM. Desktop browsers handle several GB with ease. Mobile devices may slow down above 200–300 MB.

Will hidden files (dotfiles) be included?

Browser file pickers typically expose hidden files when they are explicitly inside a selected folder. Files starting with a dot (like .gitignore) will be included if the OS allows the browser to read them.

Can I add files from multiple folders?

Yes — click "Add Files" or "Add Folder" multiple times to accumulate files from different locations before clicking Download ZIP.

How is this different from the ZIP Converter tool?

This tool focuses on creating archives with proper folder support and a file tree preview. The ZIP Converter also handles extraction.
